## 1. 研究背景与动机

### 1.1 模型结构：从单塔 ViT 到双塔 CLIP

在计算机视觉领域，模型的结构决定了其处理信息的方式：

- **传统视觉模型 (如 ViT)**: 通常可以表示为单塔函数 $f(x)$，其目标是将输入图像 $x$ 映射到一个能够区分不同类别的特征空间。
- **CLIP 模型 (视觉-语言预训练)**: 采用双塔架构，分别由图像编码器 $f_{vision}(x)$ 和文本编码器 $f_{text}(y)$ 组成。

```
┌─────────────────────────────────────────────────────────┐
│                    CLIP 架构示意                          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│    图像编码器 f_vision(x)      文本编码器 f_text(y)        │
│         ↓                          ↓                     │
│    ┌─────────┐                ┌─────────┐               │
│    │  图片 x  │ ──编码──→     │ 特征向量 │  [D维]         │
│    │ 🐱📷    │                │  v_img  │               │
│    └─────────┘                └─────────┘               │
│         ↓                          ↓                     │
│    ┌─────────┐                ┌─────────┐               │
│    │  文本 y  │ ──编码──→     │ 特征向量 │  [D维]         │
│    │ "a cat" │                │  v_txt  │               │
│    └─────────┘                └─────────┘               │
│         │                          │                     │
│         └──────────┬───────────────┘                     │
│                    ↓                                     │
│              相似度计算                                   │
│         s = v_img · v_txt / (||v||·||t||)              │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**对比学习目标**：通过最大化正样本对的余弦相似度，将图像和对应的文本描述映射到同一个对齐的特征空间。

$$
\mathcal{L} = -\frac{1}{N}\sum_{i=1}^{N} \left[ \log\frac{\exp(s_{ii}/\tau)}{\sum_j \exp(s_{ij}/\tau)} + \log\frac{\exp(s_{ii}/\tau)}{\sum_j \exp(s_{ji}/\tau)} \right]
$$

### 1.2 CLIP 的核心作用、价值与应用场景

CLIP 不仅仅是一个分类器，它通过大规模图文对齐获得了极其广泛的通用性，成为了当前多模态领域的基石模型：

- **通俗理解**: 它就像一个"**会读图的文字大师**"。它不只是识别图片里有“猫”，它理解的是“猫在沙发上睡觉”这个**概念**。
- **零样本学习 (Zero-shot)**: 无需针对特定任务训练，仅通过文本描述（Prompt）即可执行分类，能够识别从未在训练集中出现的类别。
- **广泛的工业与学术应用**: 
  - **图文检索 (Image-Text Retrieval)**: 在海量图库中通过自然语言搜索图片，如电商商品搜索、相册内容查找。
  - **生成模型指导 (Generative Models)**: 作为 DALL-E 2、Stable Diffusion 等图像生成模型的评估器或条件引导模块。
  - **开放世界物体检测 (Open-Vocabulary Detection)**: 辅助检测模型识别预定义类别之外的新物体。
  - **视频理解与动作识别**: 将帧特征与文本描述对齐，用于视频片段检索和动作分类。

### 1.3 持续学习的特殊挑战：保护预训练对齐结构

CLIP 的持续学习与传统 ViT 存在本质区别：

1. **预训练能力的保留**: 传统 ViT 的持续学习核心目标是使网络快速适应下游任务，通常允许预训练特征发生较大偏移。
2. **对齐结构的脆弱性**: CLIP 的核心价值在于 $f_{vision}$ 与 $f_{text}$ 之间精妙的几何对齐结构。对任意一塔编码器的微调，都极易破坏这种模态间的几何关联。
3. **知识丧失的代价**: 一旦对齐结构受损，CLIP 将丧失其最引以为傲的零样本泛化能力和图文检索能力。

**结论**: 这要求我们在对 CLIP 进行持续学习时，必须引入比传统 ViT 更强的**预训练知识保护机制**，防止在“学新任务”的过程中破坏这种跨模态的对齐“几何”。

> **核心矛盾**：如何让 CLIP 既学好“已见任务”，又不破坏这种宝贵的“跨模态几何结构”？

## 2. 零样本学习是什么

### 2.1 概念解释

**传统监督学习**：
```
训练:  [猫图片] → 标签"猫"     [狗图片] → 标签"狗"
预测:  新图片 → 模型知道是"猫"或"狗"
问题:  只能识别训练时见过的类别
```

**零样本学习（Zero-Shot Learning）**：
```
训练:  学习"猫"和"狗"的一般特征
预测:  给出"兔子"的描述 → 模型从未见过兔子图片
       但能通过文字描述理解并识别兔子！
```

### 2.2 CLIP的零样本分类

CLIP的零样本分类不需要训练：

```
类别标签: ["airplane", "car", "bird", "cat", "deer", ...]

模板: "a photo of a {class}"

文本编码: "a photo of an airplane" → v_txt1
         "a photo of a car"       → v_txt2
         ...

输入图片 → 图像编码 → v_img

预测 = argmax(similarity(v_img, v_txt_i))
```

**核心优势**：无需训练数据，就能识别**任意类别**（只要有文字描述）

## 5. 技术详解

### 5.1 训练端：CLIP主干的抗遗忘微调 (LoRA-NSP)

针对CLIP模型在下游任务适配过程中的灾难性遗忘问题，本研究提出了一种融合零空间参数化与低秩适应的微调方法——**LoRA-NSP（Low-Rank Adaptation with Null Space Parameterization）**。

#### 5.1.1 核心思想：零空间约束

**标准LoRA**：

$$
W = W_0 + B \times A
$$

- 更新方向 $B \times A$ 可能与预训练关键知识冲突
- 导致灾难性遗忘

#### 5.1.2 我们的方案：零空间投影

**LoRA-NSP**：

$$
W = W_0 + B \times A \times P
$$

其中 $P$ 是**投影矩阵**，将更新限制在"安全方向"：

```
┌─────────────────────────────────────────────────────────┐
│                  零空间约束示意                           │
├─────────────────────────────────────────────────────────┤
│                                                          │
│   预训练知识的"重要方向"                                  │
│         ↑                                                │
│         │    ┌─────────────────┐                        │
│         │    │   允许更新区域    │ ← 与重要方向正交        │
│         │    │   (零空间)        │                        │
│         │    │                 │                        │
│         │    │      · → ΔW     │  参数更新 ΔW            │
│         │    │                 │                        │
│         │    └─────────────────┘                        │
│         │                                                │
│    ─────┼────────────────────────────→                  │
│         │            重要方向1                           │
│                                                          │
│  核心思想: 只在不影响预训练知识的方向上更新参数              │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**投影矩阵计算**：

1. 提取每层输入特征的协方差：$\Sigma = \frac{1}{N}X^T X$
2. 特征分解：$\Sigma = V \Lambda V^T$
3. 软投影权重：$w_i = \frac{1}{1 + \beta \cdot \log(1 + \lambda_i^p)}$
4. 投影矩阵：$P = V \cdot \text{diag}(w) \cdot V^T$

#### 5.1.2 复合蒸馏机制

与此同时，本研究设计了跨模态蒸馏与模态内特征蒸馏相结合的复合蒸馏损失，进一步约束CLIP在参考数据集上的表征稳定性。

| 蒸馏类型 | 作用 | 公式 |
|---------|------|------|
| **特征蒸馏 (FD)** | 保持视觉特征稳定，模态内约束 | $\mathcal{L}_{FD} = \|f(x) - f_0(x)\|^2$ |
| **跨模态蒸馏 (CD)** | 保持视觉-文本对齐，跨模态约束 | $\mathcal{L}_{CD} = D_{KL}(p_0\|\|p)$ |

总损失：

$$
\mathcal{L}_{total} = \mathcal{L}_{CE} + \lambda_{FD}\mathcal{L}_{FD} + \lambda_{CD}\mathcal{L}_{CD}
$$

> **技术协同**：LoRA-NSP与知识蒸馏相互协同，共同实现CLIP模型对下游任务的适配与其通用能力的保持。LoRA-NSP通过在参数更新过程中引入零空间约束，从参数更新空间层面有效限制对预训练关键知识的干扰；蒸馏从输出表征层面保持稳定性。该部分的具体实现详见 `main.py` 及相关文件。

### 5.2 推理端：集成分类器的构建

#### 5.2.1 零样本分类器的局限

经上述微调后，CLIP的零样本分类器在下游任务上的性能获得显著提升。然而：
- 零样本分类器仅依赖视觉-文本编码的相似度进行决策
- 未能充分利用下游监督数据集中蕴含的视觉细粒度信息
- ID任务性能有提升空间

#### 5.2.2 LR-RGDA：低秩分解正则高斯判别分析

为此，本研究提出了一种融合低秩分解正则高斯判别分析（Low-Rank Factorized Regularized Gaussian Discriminant Analysis, LR-RGDA）与零样本分类器的集成分类框架。

**核心思想**：基于类别统计分布构建分类器


```
┌─────────────────────────────────────────────────────────┐
│              LR-RGDA分类器构建流程                        │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  训练数据                                                │
│    ↓                                                     │
│  特征提取 → 每个类别的均值和协方差                         │
│    ↓                                                     │
│  ┌─────────────────────────────────────────────────┐    │
│  │  类别1: μ₁, Σ₁    类别2: μ₂, Σ₂    类别3: μ₃, Σ₃  │    │
│  │     ·                ·                ·          │    │
│  │   /   \            /   \            /   \        │    │
│  │  ·     ·          ·     ·          ·     ·       │    │
│  │  ─────────────────────────────────────────       │    │
│  │         高斯分布: P(x|k) = N(x|μₖ, Σₖ)           │    │
│  └─────────────────────────────────────────────────┘    │
│    ↓                                                     │
│  低秩分解 + 正则化 → 判别分析 → 后验概率 P(k|x)          │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**关键特性**：LR-RGDA分类器被设计为**仅对分布内（ID）样本输出正置信度，而对分布外（OOD）样本输出零置信度**。这一特性源于LR-RGDA作为监督分类器的本质——其决策边界完全由ID数据的统计分布决定，OOD数据落在所有类别的高斯分布之外，自然获得零后验概率。

**集成分类器**：

该框架通过贡献系数$\alpha$动态调节两类分类器的权重分配。

$$
P_{ensemble} = \alpha \cdot P_{LR\text{-}RGDA} + (1-\alpha) \cdot P_{zeroshot}
$$

**实验表明**，该集成分类器在ID数据上的性能显著优于纯零样本分类器，有效提升了CLIP模型的工业部署价值。该部分的演示代码详见 `demo_ood.ipynb`。

### 5.3 推理端：自适应路由分类器

#### 5.3.1 问题：集成分类器的OOD性能仍下降

尽管上述集成分类器显著增强了ID性能，但由于LR-RGDA对OOD样本输出零置信度，导致集成分类器在OOD数据上的性能相较于零样本分类器出现退化。

这一性能损失与本文的研究目标相悖——即期望CLIP经微调后在下游任务性能提升的同时，**尽可能保持其OOD数据的零样本分类能力**（此为CLIP区别于传统视觉分类器的核心优势）。

#### 5.3.2 解决方案：基于LR-RGDA的自适应路由

为应对这一挑战，本研究进一步提出基于LR-RGDA的OOD检测器，并据此构建自适应路由分类器。

具体而言，OOD检测器首先判定输入样本的分布属性：
- 若判定为OOD样本，则激活零样本分类器进行决策
- 若判定为ID样本，则激活集成分类器进行决策

```
┌─────────────────────────────────────────────────────────┐
│                  自适应路由决策流程                        │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  输入样本 x                                              │
│      │                                                   │
│      ▼                                                   │
│  ┌─────────────────────────┐                             │
│  │ OOD检测器 (基于LR-RGDA)   │                             │
│  │ score = 1 - max(P_post) │                             │
│  └───────────┬─────────────┘                             │
│              │                                           │
│              ▼                                           │
│       score ≥ threshold?                                 │
│        /           \                                     │
│     YES             NO                                   │
│     /                 \                                  │
│    ▼                   ▼                                 │
│  ┌─────────┐      ┌────────────────────────┐             │
│  │ OOD样本  │      │ ID样本                  │             │
│  │ 激活:    │      │ 激活:                   │             │
│  │ 零样本   │      │ 集成分类器              │             │
│  │ 分类器   │      │ (LR-RGDA + Zero-shot)  │             │
│  └────┬────┘      └───────────┬────────────┘             │
│       │                       │                          │
│       └──────────┬────────────┘                          │
│                  │                                       │
│                  ▼                                       │
│               最终预测                                    │
│                                                          │
│  关键优势:                                               │
│  • OOD判定时仅切换分类器分支，无须重新执行主干前向传播          │
│  • 计算开销极为有限，实现动态决策与性能平衡                    │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

该部分的演示代码详见 `demo_ood.ipynb`。

## 6. 创新总结与拓展性

综上所述，本研究从训练端与推理端协同切入，系统提升了CLIP持续学习的有效性。

### 6.1 核心创新点与技术体系

- **训练端**：以 LoRA-NSP 和跨模态蒸馏等技术为核心，实现对模型微调的约束与优化。
- **推理端**：以基于 LR-RGDA 的自适应路由分类器（OOD检测路由 + 集成分类器）为核心，实现动态决策与性能平衡。

### 6.2 正交性与独立应用价值

这两类创新是**正交的**，具备独立应用于不同场景的灵活性：

| 创新维度 | 核心技术 | 作用机制 | 独立性 |
|:-------:|:--------|:--------|:------:|
| **训练端** | LoRA-NSP + 跨模态/模态内蒸馏 | 约束微调过程，缓解灾难性遗忘 | ✅ 可独立应用于任何ViT微调场景 |
| **推理端** | LR-RGDA集成分类器 | 融合监督学习与零样本能力 | ✅ 可独立应用于预训练CLIP模型 |
| **推理端** | 自适应路由分类器 | 动态决策，平衡ID/OOD性能 | ✅ 即使不微调CLIP也能带来增益 |

**广泛适用性**：
- **特别地**，实验表明自适应路由分类器即便在不微调CLIP的情况下，仍可为持续学习性能带来可观增益。
- **拓展性**，LoRA-NSP方法在 ViT 类增量学习、大语言模型（LLM）领域适应等任务中亦展现出广泛的适用潜力。
